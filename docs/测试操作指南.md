## 四、关键改动说明

### 改动 1：V3 混合模型去残差连接（HybridV3）

**修改文件**: [src/dl_improved.py](src/dl_improved.py)

**问题**: 原代码中 `fused = lstm_out + tf_out` 导致 Transformer 梯度冲突。
**修复**: 改为纯顺序处理，LSTM 输出 → Transformer → AttentionPooling。

```python
# 改前（梯度冲突）
lstm_out, _ = self.lstm(x)
x_pos = self.pos_encoder(lstm_out)
tf_out = self.transformer(x_pos)
fused = lstm_out + tf_out          # ← 残差连接，梯度互相干扰
pooled = self.pool(fused)

# 改后（顺序架构）
lstm_out, _ = self.lstm(x)
x_pos = self.pos_encoder(lstm_out)
tf_out = self.transformer(x_pos)   # LSTM 作为 Transformer 的前置处理器
pooled = self.pool(tf_out)         # 只对 Transformer 输出做注意力池化
```

## 五、V3 架构总览

```
Input (51×n_features)
    ↓
Conv1d(5×5) + BN + ReLU    (n_features → 32)
    ↓
Conv1d(3×3) + BN + ReLU    (32 → 64)
    ↓ permute(0,2,1)
BiLSTM                       (64 → hidden_dim*2)
    ↓
PositionalEncoding
    ↓
TransformerEncoder × 2      (hidden_dim*2 → hidden_dim*2)
    ↓
AttentionPooling             (全局注意力池化)
    ↓
FC: Linear → LayerNorm → ReLU → Dropout → Linear
    ↓
4-class output
```

三个变体：
| 模型 | Conv | LSTM | Transformer | AttentionPooling |
|------|------|------|-------------|-----------------|
| HybridV3 | ✅ | ✅ | ✅ | ✅ |
| LSTMOnlyV3 | ✅ | ✅ | ❌ | ✅ |
| TransformerOnlyV3 | ✅ | ❌ | ✅ | ✅ |

## 九、常见问题

**Q: 为什么 V3 混合不如 Transformer-Only？**
A: V3 的 Conv1d(5×5) + Conv1d(3×3) → BiLSTM 构成信息瓶颈。Conv 只提取局部模式（kernel 5+3），LSTM 进一步压缩，Transformer 接收到的序列特征可能已丢失部分全局信息。Transformer-Only 在 Conv 后直接接 Transformer，全局建模更充分。

**Q: 能不能加速评估？**
A: 快速模式已经是 `--quick`（300 棵树 + 100 epoch）。继续加速可以改 `--quick` 的默认参数（在 evaluate.py 中将 tree_n_estimators 改小）。

**Q: 结果不稳定怎么办？**
A: 深度学习模型训练有随机性，每次结果会略有波动。可以尝试固定随机种子或跑多次取平均。
