### [P2] 7. DP算法描述 — 从完整DP改为贪心枚举+Top-K搜索

#### 实际情况

`dp_post_process` 的实际实现：
```python
def dp_post_process(well_data, model_proba, prior_info=None):
    K = 10  # 每类保留top-K候选
    candidates = {1: [], 2: [], 3: []}
    
    # 为每种关键点选取top-K个高置信度位置
    for kp in [1, 2, 3]:
        kp_proba = model_proba[:, kp]
        top_k_indices = np.argsort(kp_proba)[-K:]
        ...
    
    # 贪心搜索最优组合（枚举top-5候选）
    best_score = -1
    for kp1_idx, kp1_score in candidates[1][:5]:       # 增斜点枚举top-5
        for kp2_idx, kp2_score in candidates[2][:5]:   # 稳斜点枚举top-5
            if kp2_idx < kp1_idx + 20:  # 最小间距20测点
                continue
            for kp3_idx, kp3_score in candidates[3][:5]:
                if kp3_idx < kp2_idx + 20:
                    continue
                ...
```

实际实现是 **贪心Top-K枚举搜索**，不是论文所述的完整动态规划。

实际约束参数：**MIN_GAP = 20** 测点（论文说 L_min=5），此外还有一个通用实现 `run_enhanced_dp_logic`（show.py）中使用最小间距也是20。

#### 论文需修改

**第29页**：
- ❌ `"设置L_min=5个测点，对应约10-15米最小间距"` → ✅ **修改为 "L_min=20个测点"**
- ❌ `"使用增强状态(s,d)完整编码"` → ✅ **删除，改为描述Top-K枚举搜索**

**算法描述修改为**：
> **关键点序列搜索（Top-K置信度枚举法）**：
> 
> 1. **候选选取**：对每类关键点（增斜点/稳斜点/降斜点），从模型预测概率中选取置信度最高的 K=10 个位置作为候选，要求置信度>0.1。
> 
> 2. **物理特征检查**：为每个候选点计算物理特征评分，包括井斜角变化趋势一致性（增斜点要求后续井斜上升、稳斜点要求井斜变化小、降斜点要求后续井斜下降）。
> 
> 3. **先验加权**：如果存在设计轨迹先验信息（通过DTW对齐获得），候选点离先验位置越近得分越高。
> 
> 4. **总得分**：total_score = 模型置信度 + 物理特征评分 + 先验评分
> 
> 5. **序列搜索**：对每类候选点的Top-5高分位置进行枚举，施加顺序约束（增斜→稳斜→降斜）和最小间距约束（L_min=20测点），选择总得分最高的合法组合。
> 
> 6. **失败处理**：若未找到合法组合，分别取每类最高置信度位置作为关键点。

**额外的修改**：show.py 中的 `run_enhanced_dp_logic` 使用质心法（非朴素最大值），可在论文中说明这一改进：
> "为提高关键点定位精度，采用质心法替代最大值法：在置信度峰值附近±5窗口内，以置信度为权重计算加权平均位置。"
